Web development: types of programming language and more information about the frontend

About Web-Developing

Types of programming

1 - Front-end
2 - Back-end
3 - Full-stack

Front-end
Frontend web development, also called client—side development, is the creation of websites using programming languages or CMS. The interface developer is responsible for coding and programming the website elements visible to site visitors. Front-end developers and web designers often work together to create designs and interfaces for specific brands.
For example: Text, buttons, comment field

Back-end
Backend development - sometimes called server—side development — usually involves complex programming languages such as PHP, Java, Node.js and SQL. The backend developers are responsible for the more technical aspects of the website that take place "behind the scenes" and are invisible to site visitors.
For example: Working buttons, sending a comment to the server, changing the theme of the site when you click on the button

Full-stack
Fullstack developers are perhaps the most in-demand. They write code for the external and internal interface of the site. They are well versed in how both components work separately and together. Companies that need a website created from scratch can contact the full-stack developer.

The website development process includes 6 stages:
1.Task Discussion stage - Developers collaborate with web designers and the client to determine the purpose, objectives, target audience, services and functions of the website.

2.Content Stage - Web developers use the data obtained during the task discussion stage to create a roadmap for the website, which, in fact, is a plan for all the main topics that will be covered on the site.

3.The design stage - the two most important aspects of web design are the target demographic and the function of the site. For example, a social networking site for young people should be very different from an online store aimed at older men.

4.Development Phase - Web developers use the information they collect - along with their skills — to create a working website.

5.The launch stage - the developers check for all the details, including taking into account customer feedback. They evaluate its performance and check whether the site is displayed on smartphones and desktop computers. When everything is ready, the site code is uploaded to the server for publication. Then the site is launched and becomes available for public viewing.

6.The maintenance phase - the launch of the site — is not the end of the road. A live website needs regular maintenance to keep it running at full capacity.
